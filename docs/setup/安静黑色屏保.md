# 安静黑色屏保（方案 A）

## 1. 背景
LG 电视在 **失去输入信号**（No Signal）时会进入内置的幻灯片/屏保，而且该行为在电视端无法关闭。  
在 PC 作为信号源的场景下，如果 Linux 端触发了 **DPMS 关屏 / 自动休眠**，电视就会把它当成“无信号”，从而进入幻灯片。

## 2. 目标与非目标

### 目标（本方案解决什么）
- **纯黑**：闲置后显示全屏纯黑，视觉上等价于“屏幕关了”，但仍然输出 HDMI 信号。
- **不断信号**：黑屏期间 **禁止 DPMS off / suspend**（避免电视进入幻灯片）。
- **不锁屏**：黑屏不是锁屏，不会出现密码输入界面。
- **仅自动**：只靠“闲置 10 分钟自动触发”，不额外增加手动快捷键入口（可后续扩展）。

### 非目标（本方案暂不做什么）
- 不做电视的深度控制（CEC / 网络开关机等）。
- 不强行修改 GPU 风扇曲线/超频等高风险内容。
- 仅以“单屏 HDMI 输出”为默认前提（多屏可后续扩展）。

## 3. 核心机制
本方案采用 **“黑色覆盖层（Overlay）”**：
- 闲置达到阈值后弹出一个 **全屏纯黑窗口** 覆盖桌面。
- 任何键盘/鼠标输入都会立刻退出黑屏，恢复桌面。

注意：这不是 DPMS 关屏，也不是系统休眠，所以 **HDMI 信号持续存在**，电视不会误判为“无信号”。

## 4. 关键约束与风险
- 必须关闭/禁用：
  - **Turn off screen / DPMS**（会让电视进入无信号态）
  - **Suspend / Sleep**（同样会断信号）
  - **自动锁屏**（用户偏好：黑屏不锁屏）
- 安全性取舍：黑屏不锁屏意味着离开电脑时不会自动上锁；如果你需要安全性，应改用锁屏方案（本方案不覆盖）。
- Wayland 下 `org.freedesktop.ScreenSaver.GetSessionIdleTime()` 在当前环境返回 `NotSupported`。因此闲置检测改为读取 `/dev/input/event*`（evdev）事件流来判断“是否有输入”。

## 5. 使用方式

### 启用（自动随登录启动）
```bash
systemctl --user enable --now quiet-black-screensaver.service
```

### 停用
```bash
systemctl --user disable --now quiet-black-screensaver.service
```

### 查看运行状态
```bash
systemctl --user status quiet-black-screensaver.service --no-pager
```

### 查看日志
- 文件日志（守护进程写入）：
```bash
tail -n 200 ~/.local/state/quiet-black-screensaver.log
```
- systemd 日志：
```bash
journalctl --user -u quiet-black-screensaver.service -n 200 --no-pager
```

## 6. 验收清单
1. **10 分钟无操作**：自动进入全黑。
2. **任意输入**：立即恢复桌面画面。
3. **电视不进幻灯片**：黑屏状态下仍保持当前 HDMI 输入源，不出现“无信号幻灯片”。
4. **不自动锁屏**：不会跳到锁屏界面。
5. **闲置 1 小时**：仍然是黑屏，且系统不休眠、不 DPMS 关屏。
6. 守护进程异常退出时：最多失去“自动黑屏”，但不会造成显示链路断开或其他副作用。

## 7. 回滚方式
1. 停止并禁用服务：
```bash
systemctl --user disable --now quiet-black-screensaver.service
```
2. （可选）删除脚本与 unit：
```bash
rm -f ~/.local/bin/quiet-black-screen.py
rm -f ~/.local/bin/quiet-black-screensaver-daemon.py
rm -f ~/.config/systemd/user/quiet-black-screensaver.service
systemctl --user daemon-reload
```
3. （可选）恢复 KDE 电源管理策略（DPMS/休眠/锁屏）到你原来的设置。
